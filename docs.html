CLI

npm install
node cli.mjs <image> [--model clip|siglip|dinov2] [--json]

Example:

node cli.mjs cat.jpg --model dinov2
node cli.mjs *.jpg --model clip --json > results.json

Output: color-coded bars with per-transformation similarity, plus summary stats (rotation avg, crop avg, flip similarity).

Gemma 4 Vision Encoder

The Gemma 4 vision encoder runs via ONNX Runtime Web (not Transformers.js) because it's a submodel extracted from the Gemma 4 E2B model, not a standalone pipeline model. The quantized (4-bit) ONNX build is only 112MB.

Loaded directly from onnx-community/gemma-4-E2B-it-ONNX. Image preprocessing: resize to 224×224, normalize with mean=0.5/std=0.5 (SigLIP-compatible). Output: 280 soft tokens mean-pooled to 768-dim.

Service worker

Model blobs (~85–370MB) are cached via service worker (Cache API). Network-first for app shell so updates appear immediately. Update detection with reload banner.

Key finding

Patch size is the dominant factor for transformation invariance. CLIP (patch 32) is the most invariant (83% rotation similarity). SigLIP/DINOv2 (patch 16/14) are rotation-sensitive (~30%). No encoder captures true semantic invariance the way text embeddings do.
